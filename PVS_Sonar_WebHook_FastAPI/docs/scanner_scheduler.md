# Запуск scanner.py через Планировщик Задач Windows

## Обзор

`scanner.py` теперь работает в двух режимах:
1. **Прямой доступ к БД** (старый способ, для локального запуска)
2. **Отправка через REST API** (новый способ, для запуска от имени sast_bot)

## Настройка

### 1. Добавьте настройки в .env

```env
# URL API сервера (куда scanner.py будет отправлять результаты)
API_BASE_URL=http://localhost:8080

# Ключ проекта SonarQube (используется для связи файлов с проектом)
SCAN_PROJECT_KEY=your.project.key
```

### 2. Создайте задачу в Планировщике

1. Откройте **Планировщик задач** (Task Scheduler)
2. Создайте **простую задачу**
3. Настройте триггер (например, ежедневно в 02:00)
4. В действии укажите:
   - **Программа/скрипт**: `python.exe`
   - **Аргументы**: `c:\path\to\app\scanner.py`
   - **Рабочая папка**: `c:\path\to\PVS_Sonar_WebHook_FastAPI`

5. На вкладке **Общие**:
   - ✅ **Выполнять для всех пользователей**
   - ✅ **Выполнять с наивысшими правами**
   - **Выполнять от имени**: `ARQA\sast_bot`

### 3. Проверка работы

После первого запуска проверьте логи:

**В логе scanner.py:**
```
📤 Отправка 15 результатов на http://localhost:8080/api/scan-results
✅ Результаты успешно отправлены: {'status': 'success', 'message': 'Processed 15 files'}
```

**В логе FastAPI:**
```
📥 Получены результаты сканирования: 15 файлов
✅ Добавлена версия 1.0.0.0 для file1.dll
✅ Добавлена версия 2.1.0.0 для file2.dll
✅ Результаты сканирования успешно обработаны
```

## Формат данных

### Запрос (от scanner.py)
```json
{
    "files": [
        {
            "project_key": "project.key",
            "filename": "file.dll",
            "version": "1.0.0.0",
            "file_hash": "abc123..."
        }
    ]
}
```

### Ответ (от API)
```json
{
    "status": "success",
    "message": "Processed 15 files"
}
```

## Устранение неполадок

### Ошибка: "Не удалось отправить результаты через API"

**Причины:**
- FastAPI сервер не запущен
- Неправильный URL в `API_BASE_URL`
- Блокировка фаерволом

**Решение:**
1. Проверьте что сервер доступен: `curl http://localhost:8080/docs`
2. Проверьте логи FastAPI
3. Проверьте что порт 8080 не заблокирован

### Ошибка: "Проект не найден"

**Причины:**
- Неправильный `SCAN_PROJECT_KEY` в .env
- Проект не создан в БД

**Решение:**
1. Проверьте `SCAN_PROJECT_KEY` в `.env`
2. Создайте проект через веб-интерфейс `/`

### Ошибка: "Файл не имеет версии"

**Причины:**
- Файл не имеет версии в метаданных
- Ошибка при чтении версии

**Решение:**
- Это нормальная ситуация для некоторых файлов
- Такие файлы пропускаются с предупреждением в логе

## API Endpoint

### POST /api/scan-results

Принимает результаты сканирования от scanner.py.

**Тело запроса:**
```json
{
    "files": [
        {
            "project_key": "string",
            "filename": "string",
            "version": "string",
            "file_hash": "string"
        }
    ]
}
```

**Ответ:**
- `200 OK`: `{"status": "success", "message": "Processed N files"}`
- `500 Error`: `{"detail": "error message"}`
